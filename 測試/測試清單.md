# Trip Planner 第三輪完整功能測試

## 專案概述
使用 React 19 + Vite + Tailwind CSS + Supabase 建構的旅程規劃工具。
- Dev Server 啟動指令：`npm run dev`
- 專案入口URL：`http://localhost:3002/tokyo-trip/`（如果 3002 被佔用，請看 terminal 輸出的實際 port）

## 應用架構
App 透過 state 切換三個頁面：
1. **AuthPage**（登入/註冊頁）
2. **TripList**（行程列表頁）— 含行程範本卡片
3. **TripDetail**（行程詳細頁）— 含 4 個分頁：**行程 | 地圖 | 預算 | 打包清單**

## 本輪測試範圍

| 功能模組 | 說明 |
| :--- | :--- |
| 行程範本 | 空狀態下一鍵套用完整行程 |
| 拖曳排序 | 事件卡片拖曳重新排列 + 跨日移動 |
| 智慧分類 | 標題關鍵字自動推斷事件類型 |
| 互動地圖 | Leaflet 地圖標記景點 + 路線連線 |
| 預算追蹤 | 花費記錄、幣別切換、分類統計 |
| 打包清單 | 物品管理、勾選、預設清單 |
| 核心功能 | CRUD、localStorage 持久化、回歸測試 |

## 完整使用情境

> **角色**：小明是一位大學生，想跟朋友規劃一趟東京旅行。他第一次使用 Trip Planner，從零開始到行程完整規劃完畢。

### 階段一：首次進入 & 新手體驗

#### 測試 1：登入頁面基本檢查
1. 開啟應用，確認顯示「Trip Planner」標題與登入表單
2. 確認有 Email 輸入框、密碼輸入框、登入按鈕、Google 登入按鈕
3. 確認有「跳過登入，使用本地模式」連結
4. 點擊「跳過登入，使用本地模式」→ 確認進入 TripList 頁面

#### 測試 2：空狀態 & 行程範本顯示
1. 確認頁面顯示「還沒有任何行程」提示
2. 確認有「從頭建立」按鈕
3. 確認下方出現「一鍵套用範本」標題（帶 ✨ 圖示）
4. 確認顯示 3 張範本卡片：
   - 「東京五天四夜初體驗」（有漸層色標題欄、天數、景點數、預算金額）
   - 「首爾三天兩夜購物行」
   - 「大阪京都四日遊」
5. 確認每張範本卡片 hover 時有陰影變化

#### 測試 3：一鍵套用範本
1. 點擊「東京五天四夜初體驗」範本卡片
2. 確認自動跳轉到 TripDetail 頁面
3. 確認 Hero Header 顯示：
   - 標題：「東京五天四夜初體驗」
   - 描述：「經典東京必去景點 + 美食購物行程」
   - 日期範圍（應為一週後起算 5 天）
   - 「本地存儲」標籤
4. 確認導航列出現 Day 1 ~ Day 5 共 5 個日程標籤
5. 確認 Day 1 的主題為「抵達 & 新宿探索」，包含 4 個事件
6. 切換到 Day 3 → 確認主題為「原宿 & 澀谷」，包含 5 個事件
7. 確認每個事件都有正確的時間、標題、類型圖示

#### 測試 4：範本預算資料驗證
1. 點擊「預算」分頁
2. 確認全程總花費顯示合理金額（¥ 開頭）
3. 確認「5 天」與事件數量顯示正確
4. 確認「每日花費」列出 Day 1 ~ Day 5 各自的花費
5. 確認「分類花費」至少包含交通、食物、購物、景點等分類

#### 測試 5：範本打包清單驗證
1. 點擊「打包清單」分頁
2. 確認已預填 13 項常用物品（護照、身分證、機票/電子票…）
3. 確認完成計數顯示「0/13 已完成」
